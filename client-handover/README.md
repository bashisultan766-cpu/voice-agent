# Client handover package — AI Voice Agents for Bookstores

This folder contains the full handover set for launch and ongoing operations. Use it for client delivery, onboarding, and support.

---

## Contents

| Folder / file | Description |
|----------------|-------------|
| **01-product-overview** | What the system does, who it’s for, main modules, limitations. |
| **02-admin-manual** | How to use the dashboard: stores, agents, knowledge, calls, analytics, QA. |
| **03-technical-architecture** | Architecture, stack, deployment, secrets, backups, integrations. |
| **04-env-and-integrations** | Env vars and third-party accounts (Twilio, Shopify, OpenAI, etc.). |
| **05-sops** | SOPs: new store onboarding, new agent, prompt update, Shopify reconnect, escalation handling. |
| **06-support-runbooks** | Incident runbooks: calls not answered, wrong store info, order lookup failing, too many escalations. |
| **07-access-inventory** | Access inventory template and handover checklist. |
| **07-pilot-rollout-plan.md** | Phased pilot rollout (weeks 1–4). |
| **08-go-live-checklist** | Go-live checklist (product, ops, business, rollout, sign-off). |
| **09-roadmap-v2** | v2 roadmap: multilingual, callback automation, CRM, WhatsApp, analytics, human handoff, autonomous workflows. |
| **10-maintenance-plan** | Maintenance and support proposal: scope, tiers, exclusions. |

---

## Delivery checklist

- [ ] All documents reviewed and customized (client name, URLs, contacts).
- [ ] Access inventory filled and signed off.
- [ ] Go-live checklist completed and sign-off obtained.
- [ ] Pilot rollout plan agreed and dates set.
- [ ] Support levels (L1/L2/L3) and contacts documented.
- [ ] Maintenance tier (if any) agreed and documented.
- [ ] Handover session(s) completed (admin basics, content ops, monitoring, go-live).
- [ ] Client has access to this folder (or exported pack) and knows where to find runbooks and SOPs.

---

## Document versions

Keep a note of the handover date and any doc version (e.g. v1.0 – March 2025). Update runbooks and SOPs when processes or platform behavior change.
